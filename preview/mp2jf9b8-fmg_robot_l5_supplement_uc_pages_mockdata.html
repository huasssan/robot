0. Scope Contract

补充结论：这次补充引入 3 个下钻详情页，但不新增一级导航。它们是为了让今天讨论明确覆盖 UC1、UC2 和生产计划变化巡检，不改变现有 L5 的一级页面结构。
项 结论 说明
是否改变一级页面树 否 仍以 Command Center、Mission、Spatial、Anomaly、Emergency、Skill、Robot、Evaluation、Visual Data 为主体。
是否新增页面职责 是 新增 3 个详情页：On-site Anomaly Task Detail、Weather Emergency Dispatch Detail、Production Plan Change Inspection Detail。
是否改变安全口径 需要确认 UC2 原始想法偏全自动，L5 建议改成 auto-dispatch candidate + HITL gated，避免承诺无人审批的高风险物理处置。
native HTML 要补什么 补 mock 数据和下钻视图 至少补 3 个 attention item、3 个详情对象、对应 route/link、页面渲染函数或页面状态卡片。

3. 页面职责说明表

页面名称 页面类型 页面目标 关键输入 关键输出 关键操作 进入条件 跳转去向
On-site Anomaly Task Detail 现场巡检异常详情页 让操作员逐个处理机器狗巡检中发现的异常，完成靠近复查、证据采集、报告生成和后续任务创建。 PatrolMission、PatrolAnomaly[]、EvidenceRef[]、RobotState、ControlPolicy。 PatrolReportDraft、FollowupMissionDraft、EvidencePackage、OperatorDecision。 Select anomaly、Inspect Closer、Capture Evidence、Pause Patrol、Hold Position、Create Follow-up Mission。 机器狗执行巡检 mission，且现场产生至少一个 anomaly。 Mission、Robot Profile、Visual Data、Spatial。
Weather Emergency Dispatch Detail 极端天气应急派发详情页 把天气阈值和设备 CRITICAL 告警转成应急 mission candidate，并明确机器人匹配、物理动作步骤和 HITL 安全边界。 WeatherRiskEvent、CriticalEquipmentAlert、EmergencyMission、SkillMatch、RobotMatch、SafetyGate。 DispatchDecision、ApprovalRecord、PhysicalActionExecution、VerificationEvidence。 Approve Candidate、Change Robot、Approve Step、Set HITL Mode、Safe Stop、E-stop、Verify Result。 天气阈值超限且关键设备进入 CRITICAL 状态。 Emergency Intervention、Skill Registry、Robot Profile、Visual Data。
Production Plan Change Inspection Detail 计划变化巡检重编排详情页 解释生产计划变化如何影响巡检任务、路线、空间风险和人工决策，完成新巡检计划确认。 ProductionPlanDelta、AffectedInspectionTask[]、RoutePlan、SpatialConflictSet、SafeCorridorOption[]。 InspectionPlanChange、RouteDecision、OperatorDecision、DownstreamMissionUpdates。 Compare Versions、Recalculate Route、Select Corridor、Approve New Plan、Hold Affected Missions、Rollback。 生产计划、爆破窗口、货车优先级或区域策略发生变化，且影响未完成巡检任务。 Mission、Spatial、Command Center、Visual Data。

4. 页面与业务节点映射

页面 承载 L3 节点 回答的业务问题 为什么不能只靠现有页
On-site Anomaly Task Detail B01 事件进入、B02 AI 判断、B03 Mission 生成、B07 执行监控、B08 证据回收。 机器狗已经在现场巡检，发现多个异常后，操作员如何逐个核查、取证、暂停和生成后续任务？ 现有 Anomaly Detail 以单个 CV-12 异常为中心，缺少巡检现场产生多个 anomaly 后的队列和控制动作。
Weather Emergency Dispatch Detail B01 事件进入、B03 Mission 生成、B05 能力匹配、B06 HITL、B07 执行监控。 天气超阈值和设备 CRITICAL 告警同时发生时，系统如何推荐人形机器人执行物理动作，同时守住安全审批边界？ 现有 Emergency Detail 有执行态，但缺少 weather trigger、critical asset trigger 和 full-auto/HITL 口径决策。
Production Plan Change Inspection Detail B01 事件进入、B04 空间规划、B04A 边缘安全、B06 HITL、B07 执行更新。 生产计划变化如何影响巡检计划、机器人路线和空间安全，谁确认变更，变更后下发什么？ 现有 Spatial 页能解释路线冲突，但不能完整解释 plan delta、affected tasks、inspection plan version 和 decision ledger。

5. 页面状态矩阵

页面 Default Review / Approval Blocked / Risk Empty / Loading / Error Permission
On-site Anomaly Task Detail 显示巡检 mission、异常队列、当前 evidence 和机器人状态。 anomaly 等待 operator review，follow-up mission 待确认。 机器人进入 restricted zone 风险、PHM 降级、证据质量不足。 无 anomaly、证据加载中、media load failed、robot telemetry missing。 远控、暂停和创建 mission 需要 Site Safety Lead 或 Dispatcher 权限。
Weather Emergency Dispatch Detail 显示 weather threshold、critical alert、dispatch candidate 和物理步骤。 auto-dispatch candidate 等待 human gate；物理动作逐步审批。 天气严重到禁止派发、通信降级、机器人不满足 weather-rated skill、E-stop active。 天气源缺失、设备告警缺失、候选机器人为空、执行证据缺失。 Approve physical action、safe stop、E-stop 分别需要不同安全权限。
Production Plan Change Inspection Detail 显示 plan delta、affected tasks、route conflict 和新巡检计划。 新 plan version 待批准；safe corridor 待选择。 路线不可行、blast zone 冲突、CAS 20m 冲突、任务必须 hold。 无受影响任务、计划源加载中、route calculation failed、spatial map stale。 Approve New Plan、Rollback、Hold Missions 需要 Dispatcher 或 Site Safety Lead 权限。
